# FINAL LIVE FARM UPGRADE REFRESH

## Что исправлено

Исправлен баг, из-за которого улучшение фермы на сайте срабатывало быстро, но уровень и цены визуально обновлялись только после ручного `F5`.

## Причина

Action-endpoint `/api/farm/upgrade` отдавал обновлённый профиль, но без части полей из полного `/api/me` ответа, особенно без `user`.
Из-за этого быстрый repaint после апгрейда мог не отрисоваться корректно, и пользователь видел старый уровень до полного reload страницы.

## Что сделано

### public/app.js
Добавлен финальный live-hotfix:
- action state после апгрейда теперь мержится с текущим `state`
- сохраняется `user` и остальные поля UI, которых может не быть в action-ответе
- после успешного апгрейда сразу делается `render(...)` нового состояния
- кнопки `Улучшить ферму +1` и `Улучшить ферму +10` перевязаны на финальный live-обработчик
- затем идёт тихий `loadMe(true)` для добивки консистентности

## Результат

Теперь после клика по апу фермы:
- новый уровень должен показываться сразу на сайте
- цена следующего апа должна меняться сразу
- ручной `F5` больше не нужен
