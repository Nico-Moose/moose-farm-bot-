# FALSE STALE PROFILE + FAST UPGRADE HOTFIX

## Что исправлено

Этот hotfix убирает ложные `stale_profile` после быстрых улучшений на сайте.

### Причина бага

После быстрого апгрейда сайт сохранял профиль и сразу отдавал ответ фронту.
Затем асинхронная очередь синка в WizeBot вызывала `markWizebotSyncAt(...)`, а та функция обновляла не только `last_wizebot_sync_at`, но и `updated_at` профиля.

Из-за этого следующий клик по улучшению часто приходил с уже устаревшим `expectedUpdatedAt`, хотя игровое состояние фактически не менялось. Сервер отвечал `stale_profile`.

## Исправления

### 1. services/userService.js
- `markWizebotSyncAt(...)` больше не меняет `updated_at`
- меняется только `last_wizebot_sync_at`

### 2. public/app.js
- добавлен hotfix для кнопок `Улучшить ферму +1` и `+10`
- при `stale_profile` фронт сначала тихо применяет свежий state из ответа
- затем автоматически один раз повторяет апгрейд уже с новым состоянием
- кнопки перевязаны на новый быстрый обработчик с локальным lock

## Результат

- ложные `stale_profile` после async sync больше не должны мешать апам
- ферма обновляется визуально сразу из ответа сервера
- быстрые повторные клики работают заметно мягче

## Что не делалось в этом hotfix

- не менялась логика рейдов
- не менялась логика кейсов
- не менялась логика рынка
- не менялась серверная экономика

Hotfix точечный: только скорость/стабильность апгрейда фермы на сайте.
