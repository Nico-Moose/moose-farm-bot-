# README CASE SCROLL PERF PATCH

Дата: 2026-05-03

Что сделано:
- Убрано второе отдельное окно результата кейса. Теперь `openCase()` открывает только одну модалку рулетки, а итог показывается внутри неё.
- Кнопка «вверх» перенесена вниз вправо, сделана компактнее и визуально ближе к зелёному UI.
- Ослаблена лишняя загрузка на фронте:
  - `loadMe()` теперь использует короткий кэш `/api/me`;
  - история больше не грузится на каждый `loadMe()`, а подгружается только на вкладке журнала;
  - топы подгружаются лениво при открытии вкладки топов.

Зачем:
- убрать визуальный баг с двумя кейс-окнами;
- сделать кнопку прокрутки менее навязчивой;
- уменьшить количество лишних запросов и повторных перерисовок при открытии страницы и переключении вкладок.

Что ещё можно делать дальше:
- агрегировать несколько dashboard-запросов в один summary endpoint;
- кешировать историю и топы на сервере коротким TTL;
- проверить, не дублируются ли fetch-запросы в отдельных модалках и карточках.
