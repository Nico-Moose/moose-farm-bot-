# Stage 1.3 — safe frontend cleanup in tops/info

## Что изменено

Только узкая зона `Топ / инфо`, без касания рынка, зданий и игровых API.

Изменённые файлы:
- `public/js/09-admin-overrides.js`
- `public/js/10b-case-tabs-loader.js`
- `public/js/10f-history-raid-case-offcollect.js`

## Что сделано

1. `Топ / инфо` переведён на более чистый lazy-load:
   - топы больше не подтягиваются скрыто из `renderInfo()` когда вкладка не открыта;
   - загрузка идёт при открытии вкладки `tops` или по кнопке `Обновить топы`;
   - если вкладка не активна, `refreshTopsIfVisible()` ничего не делает.

2. Добавлены safe-helpers:
   - `isFarmTabActive(name)`
   - `refreshTopsIfVisible(force)`

3. `loadTops()` теперь защищён от устаревших ответов:
   - новый запрос отменяет предыдущий;
   - старый ответ не перетирает новый UI;
   - при ручном refresh показывается `Загрузка топов...`.

4. Кнопка `🏆 Обновить топы` теперь вешается через `onclick` после перерендера блока, без накопления legacy-обработчиков.

## Что не менялось

- рынок
- здания
- история
- API-контракты
- расчёт топов на сервере
- визуальный вид блока `Топ / инфо`

## Зачем это безопасно

Патч не меняет бизнес-логику и не меняет маршруты. Он только убирает лишние скрытые запросы и legacy-слой в отдельной фронтовой вкладке.
