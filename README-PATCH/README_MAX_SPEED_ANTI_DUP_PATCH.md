# MAX SPEED + ANTI-DUP PATCH

Что сделано:

1. **Ускорены ключевые апгрейды сайта**
   - `farm/upgrade`
   - `farm/building/buy`
   - `farm/building/upgrade`
   - `farm/raid-power/upgrade`
   - `farm/protection/upgrade`
   - `farm/turret/upgrade`

   Для этих действий сайт теперь **сохраняет профиль и сразу отвечает фронту**, а синк в WizeBot уходит **в асинхронную очередь**.

2. **Добавлена очередь синка в WizeBot**
   - новый сервис `services/wizebotSyncQueueService.js`
   - синки коалесцируются по логину
   - повторные быстрые изменения не создают длинную цепочку одинаковых пушей

3. **Добавлена защита от stale-state / дюпов**
   - фронт отправляет `expectedUpdatedAt`
   - сервер сравнивает его с актуальным `profile.updated_at`
   - если профиль уже изменился, сервер отдаёт `409 stale_profile`
   - фронт мгновенно перезагружает свежий профиль

4. **Убран лишний full reload после апов**
   - фронт теперь использует уже готовый payload из ответа action endpoint
   - после успешного апа не нужен лишний `GET /api/me`
   - это заметно ускоряет сайт на слабом интернете

Что важно:
- Этот патч специально оптимизирует **скорость улучшений и действий на сайте**.
- WizeBot всё ещё получает обновления, но уже **не блокирует ответ пользователю** на ключевых апгрейдах.
- Защита от двойных кликов и устаревшего состояния сохранена и усилена.

Что пока не переводилось в async специально:
- кейсы
- рейды
- рынок
- лицензии
- сборы

Их можно перевести следующим патчем отдельно, но сначала безопаснее проверить новые быстрые апгрейды в бою.
