# Stage 1.2 — safe frontend cleanup (history lazy-load only)

## Что сделано

Узкая зачистка фронта только в зоне истории/журнала, без касания рынка.

Изменённые файлы:
- `public/js/01-core-main.js`
- `public/js/04-app-actions.js`
- `public/js/10d-live-refresh-stale.js`

## Суть safe-patch

В проекте уже есть логика ленивой загрузки истории только на вкладке `history`
(см. переключение вкладок в `10b-case-tabs-loader.js`).
Но поверх неё оставались старые override-вызовы, которые всё равно дёргали
`loadHistory()` после обычного `loadMe()` даже когда вкладка истории скрыта.

Это создавало лишние запросы и оставляло исторический дубль поведения.

Патч делает следующее:

1. Добавлен helper `isFarmTabActive(name)`.
2. Добавлен helper `refreshHistoryIfVisible()`.
3. `refreshVisibleData()` теперь обновляет историю только если реально открыта вкладка журнала.
4. Базовый `loadMe()` в `04-app-actions.js` теперь не дёргает историю вслепую.
5. Override `loadMe()` в `10d-live-refresh-stale.js` тоже обновляет историю только если вкладка видима.
6. Удалён старый legacy-wrapper вокруг `loadHistory` в `10d-live-refresh-stale.js`, который пытался
   прокинуть в `loadHistory()` URL-аргумент, хотя базовая функция его не использует.

## Что НЕ менялось

- URL API не менялись
- рендер истории не менялся
- рынок не трогался
- здания не трогались
- админка не трогалась
- поведение при открытой вкладке `История / журнал` сохранено

## Ожидаемый эффект

- меньше лишних запросов к `/api/farm/history`
- меньше layered-override мусора в фронте
- поведение становится ближе к уже задуманной lazy-load схеме истории

## Что проверить руками

1. Открытие сайта на главной вкладке не должно автоматически ломать журнал.
2. При переходе во вкладку `История / журнал` список должен загружаться как раньше.
3. Кнопка `Обновить журнал` и фильтр типа события должны работать как раньше.
4. После обычных действий на главной/зданиях/топах скрытая история не должна спамить лишними запросами.
