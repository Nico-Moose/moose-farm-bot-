# LIVE NO-CACHE INSTANT UPGRADE REFRESH

## Что исправлено

Hotfix устраняет ситуацию, когда улучшения на сайте фактически проходят, но UI не обновляется до `F5`.

## Причина

Браузер мог держать старый ответ `GET /api/me` в кеше, а часть обработчиков после успешных апгрейдов делала именно повторный `loadMe()`.

Из-за этого получалось:
- действие прошло
- база уже обновилась
- `loadMe()` вернул старое кешированное состояние
- уровень/цены на странице не менялись до ручного `F5`

## Изменения

### routes/apiRoutes.js
Добавлены заголовки, запрещающие кеширование API-ответов:
- `Cache-Control: no-store, no-cache, must-revalidate, proxy-revalidate`
- `Pragma: no-cache`
- `Expires: 0`
- `Surrogate-Control: no-store`

### public/app.js
- `loadMe()` переведён на `fetch('/api/me?_=' + Date.now(), { cache: 'no-store' })`
- `loadHistory()` тоже получает no-cache fetch
- для кнопок `Улучшить ферму +1` и `+10` добавлен мгновенный repaint профиля из ответа action-endpoint
- после мгновенного repaint делается тихий догруз свежего профиля, чтобы UI оставался консистентным

## Эффект

Теперь после улучшения:
- уровень должен обновляться сразу на странице
- стоимость следующего апа тоже должна меняться сразу
- ручной `F5` больше не нужен для обычного обновления UI

## Что не менялось

- экономика фермы
- логика WizeBot sync
- логика рейдов/кейсов/рынка

Hotfix точечный: только живое обновление UI и защита от кеша.
