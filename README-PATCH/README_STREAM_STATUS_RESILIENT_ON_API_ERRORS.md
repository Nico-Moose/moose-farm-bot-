# PATCH: стрим не должен падать в оффлайн из-за временной ошибки Twitch API

## Что исправлено
- Если Twitch API временно возвращает ошибку (например `twitch_stream_401`), вкладка `Стрим` больше не переключается сразу в оффлайн-картинку.
- При ошибке проверки вкладка сохраняет текущий стрим или, если состояние ещё неизвестно, показывает сам player как безопасный fallback.
- Добавлен повторный опрос статуса раз в 60 секунд для активной вкладки `Стрим`.

## Изменённые файлы
- `services/streamStatusService.js`
- `public/js/10l-stream-tab.js`
