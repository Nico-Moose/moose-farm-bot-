# WizeBot 1:1 Economy Refactor Patch

Этот патч делает экономику фермы ближе к WizeBot-логике и раскладывает формулы по отдельным модулям, чтобы дальше было проще редактировать и добавлять новые функции.

## Новые модули

- `services/farm/economyConfig.js` — все важные константы WizeBot: лимиты, буферы, цены рынка, рейд/защита, турель.
- `services/farm/economyMath.js` — чистые формулы: ап фермы, лимиты зданий, стоимость зданий, защита, прогрессивные апы.
- `services/farm/walletService.js` — единая система списания денег с WizeBot-порядком и буфером 100000.
- `services/farm/incomeService.js` — доход фермы: пассив, растения, животные, здания, завод/фабрика/шахта.

## Что переработано

### Ап фермы

`upgradeService.js` теперь использует формулу WizeBot `calcCost` 1:1 и возвращает удобные поля:

- `needed.cost`
- `needed.parts`
- `needed.availableCoins`
- `needed.availableParts`
- `needed.missingCoins`
- `needed.missingParts`
- `spent`

### Балансы

`walletService.js` поддерживает разные WizeBot-режимы списания:

- `farm_upgrade`: фермерский сверх буфера → Twitch сверх буфера → бонусный → фермерский → Twitch.
- `building`: Twitch сверх буфера → фермерский сверх буфера → бонусный → фермерский → Twitch.
- `turret`: Twitch сверх буфера → бонусный → Twitch → фермерский.
- `upgrade_only`: только бонусный баланс.

Важно: реальный Twitch-баланс сайт всё ещё не может менять без обратного WizeBot API. Поэтому поле `twitch_balance` поддержано в коде, но если его нет в SQLite, оно считается `0`.

### Здания

`buildingService.js` теперь использует отдельные формулы:

- покупка здания;
- ап здания;
- лимит уровня по уровню фермы;
- ограничения шахты/фабрики;
- понятные `missingCoins/missingParts` в состоянии карточек.

### Доход

`collectService.js` теперь работает через `incomeService.js`:

- пассив `level * 4 * 0.5`;
- растения/животные по WizeBot-конфигам;
- доход зданий;
- завод даёт запчасти;
- фабрика усиливает запчасти;
- шахта добавляет процент к запчастям;
- кузница копит оружие;
- укрепления копят щит.

### Рейд-сила и защита

`raidUpgradeService.js` использует WizeBot-прогрессию:

- цена уровня: `400000 * следующий_уровень`;
- рейд-сила максимум 200;
- защита максимум 120;
- каждый уровень защиты = `0.5%`.

### Турель

`turretService.js` использует WizeBot-таблицу апов и возвращает нехватку денег/запчастей.

### Рейды

`raidService.js` вынесен ближе к WizeBot:

- выбор цели с весами богатых игроков;
- центр рейдов режет кулдаун;
- рейд-сила + уровень + оружие;
- AFK/punish multiplier;
- защита 0.5% за уровень;
- щит укреплений;
- глушилка против турели;
- штраф турели;
- бонусные монеты от завода/фабрики/шахты.

## Что ещё не 100% без WizeBot API

Полный 1:1 невозможен до обратной синхронизации сайт → WizeBot, потому что сайт пока не может реально читать/списывать Twitch currency в момент действия. Код уже подготовлен под `twitch_balance`, но нужно будет подключить WizeBot write/read API.
